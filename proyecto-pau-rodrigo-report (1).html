Línea base de apoyo a la accesibilidad
Navegadores web compatibles Google Chrome (versión más reciente) Este navegador es ampliamente utilizado y ofrece soporte robusto para estándares modernos de accesibilidad. Mozilla Firefox (versión más reciente) Firefox es conocido por su compatibilidad con tecnologías de asistencia y su adherencia a los estándares web. Microsoft Edge (versión más reciente) Edge, basado en Chromium, es similar a Google Chrome en cuanto a soporte de accesibilidad y tecnologías de asistencia. Safari (versión más reciente) Safari es el navegador predeterminado en dispositivos Apple y ofrece un buen soporte para tecnologías de asistencia como VoiceOver. Tecnologías de asistencia compatibles NVDA (NonVisual Desktop Access) con Firefox NVDA es un lector de pantalla gratuito muy utilizado en Windows y es totalmente compatible con Firefox. JAWS (Job Access With Speech) con Chrome y Edge JAWS es un lector de pantalla comercial ampliamente utilizado en entornos profesionales, y es compatible con navegadores como Chrome y Edge. VoiceOver con Safari VoiceOver es un lector de pantalla integrado en dispositivos Apple (macOS e iOS), compatible con Safari. TalkBack con Chrome (Android) TalkBack es un lector de pantalla integrado en dispositivos Android y funciona bien con Chrome.
Requisitos de evaluación adicionales
Requisitos de evaluación adicionales El informe incluye una lista completa de todos los errores identificados El informe de accesibilidad no se limitará a ejemplos, sino que incluirá una lista detallada de todos los errores encontrados en el sitio web durante la evaluación. Esto asegurará que se identifiquen y documenten todos los problemas de accesibilidad, permitiendo un análisis exhaustivo y una mejor planificación de correcciones. El informe proporcionará una descripción clara de cada problema encontrado. Cada error identificado irá acompañado de una descripción clara y detallada del problema, incluyendo su impacto en los usuarios con discapacidades y la razón por la cual no cumple con las pautas de accesibilidad WCAG. El informe incluye sugerencias de reparación para cada error listado Para cada error de accesibilidad, se proporcionarán sugerencias específicas y prácticas sobre cómo corregir el problema, garantizando que el sitio cumpla con los requisitos de accesibilidad necesarios. La evaluación cubrirá todas las páginas y el contenido del sitio web La evaluación de accesibilidad no se limitará a una selección de páginas, sino que cubrirá todas las páginas y el contenido del sitio web, asegurando que la accesibilidad se mantendrá de manera consistente en todo el sitio . Se evaluará la compatibilidad con tecnologías de asistencia en diversas plataformas. Se evaluará la compatibilidad del sitio web con las principales tecnologías de asistencia, como lectores de pantalla, y se probará el funcionamiento en diferentes plataformas y dispositivos, incluidos escritorios y móviles. La evaluación incluye pruebas con diferentes tamaños de pantalla y niveles de zoom. Se realizarán pruebas para garantizar que el sitio sea accesible y funcional cuando los usuarios cambien el tamaño de la pantalla o utilicen el zoom en sus navegadores, cubriendo al menos un 200% de zoom. . Se evaluará la accesibilidad para la navegación mediante teclado. Se llevará a cabo una prueba exhaustiva para asegurar que todos los elementos interactivos del sitio web, como botones, enlaces y formularios, sean accesibles mediante teclado, sin depender de un ratón. El informe incluye un resumen de los resultados de la evaluación Al final del informe, se incluye un resumen general que indica el cumplimiento general del sitio web con las pautas de accesibilidad WCAG, destacando los principales problemas identificados y el impacto de estos problemas en los usuarios. . La evaluación se realizará utilizando herramientas automáticas y pruebas manuales. Se combinarán herramientas automáticas de evaluación de accesibilidad con pruebas manuales para identificar problemas que no pueden ser detectados automáticamente, como problemas relacionados con el diseño visual o el contenido.
